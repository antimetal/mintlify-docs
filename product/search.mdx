Search is how you ask questions and explore your infrastructure in Antimetal. Instead of switching between monitoring tools, run a natural-language query and get answers grounded in real data from your connected platforms.

## How it works

Go to [Search](https://overlook.antimetal.com/search) and type a question about your infrastructure. Antimetal queries your connected platforms, including logs, metrics, traces, events, and code, and returns an answer with supporting evidence.

Examples of things you can search:

- "What's the p99 latency for the payments service?"
- "Are there any errors in the auth service logs from the last hour?"
- "What changed in the checkout service since yesterday?"
- "How much CPU is the API gateway using?"

## Starting an investigation from search

If search surfaces something that needs attention, you can create an issue directly from the results. This kicks off a full [investigation](/product/issue-lifecycle#the-investigation) with root cause analysis and remediation steps.

## Data sources

Search pulls from every platform you've connected to Antimetal. The more integrations you have, the more complete your results will be.

| Category | Platforms |
|----------|-----------|
| **Monitoring and Observability** | [Datadog](/integrations/datadog), [Grafana Cloud](/integrations/grafana-cloud), [Sentry](/integrations/sentry) |
| **Cloud and Infrastructure** | [CloudWatch](/integrations/cloudwatch), [GCP](/integrations/gcp), [Vercel](/integrations/vercel) |
| **Incident Management** | [PagerDuty](/integrations/pagerduty), [Incident.io](/integrations/incident-io) |
| **Source Control** | [GitHub](/integrations/github) |

To connect a new platform, see [Integrations](/integrations/overview).
